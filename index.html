บทนำ — DGL (Deep Guided Learning)

เป้าหมายของหนังสือ

เล่มนี้สร้างเพื่อ “สอนให้คิด + จับมือทำ” จนคุณอ่านตลาดได้อย่างเป็นระบบ — ไม่ใช่แค่จำรูปแบบ แต่เข้าใจเหตุผลเชิงโครงสร้างและจิตวิทยาของราคา

  • Objective: ระบุเป้าหมายให้ชัดในทุกบท
  • Process: ทำทีละขั้น ตั้งแต่มุมมองภาพใหญ่จนถึงจุดเข้าจริง
  • Observation & Why: ดูอะไร ทำไมถึงดู สิ่งนั้นสะท้อนอะไร
  • Worked Example: ใช้กราฟจริง TradingView (ฝังในเล่ม)
  • Reflection: คำถามสะท้อน—เพื่อสร้างสัญชาตญาณการตัดสินใจ
เคล็ดลับ: “อย่ารีบเข้า จนกว่าจะยืนยันด้วย โครงสร้าง + สภาพคล่อง + ค่า (Value) + เวลา ครบ 4 อย่าง”

CH01 — Market Structure (BOS/CHoCH)

Objective

กำหนด Directional Bias ด้วยหลักฐาน: BOS (แท่งปิดทำลายโครงสร้างเดิมอย่างมีแรง) และใช้ CHoCH เป็นสัญญาณเปลี่ยนมือ

Process (จับมือทำ)
  1. วาดสวิงหลัก HH/HL หรือ LH/LL ให้ชัด (ข้าม noise)
  2. รอแท่ง displacement ปิดทะลุสวิง (BOS)
  3. หลัง BOS ให้หา value ถัดไป (OB/FVG/Fib) เพื่อยืนยันจุดเข้า
  4. ย่อลง TF 15m→5m→1m เพื่อหา micro BOS เป็นทริกเกอร์
What to Observe & Why
  • Follow-through หลัง BOS — คือแรงต่อเนื่อง ไม่ใช่แค่แทงทะลุเฉย ๆ
  • BOS ใกล้ value zone (ไม่ใช่กลาง range) → เพิ่มโอกาส RR ดี
  • CHoCH = จุด “เปลี่ยนบทสนทนา” ของแรงซื้อ–ขาย
Common Mistakes
  • ตีโครงสร้างจาก noise ทำให้เห็น BOS หลอก
  • เข้าเทรดทันทีหลัง BOS โดยไม่รอ value
  • ละเลย timeframe ใหญ่จนสวนทิศภาพรวม
Reflection: “ถ้าคุณลบทุกเส้นออก เหลือเพียงสวิงและแท่งปิด — ยังยืนยัน BOS เดิมได้หรือไม่?”

CH02 — Liquidity & Inducement

Map สภาพคล่อง
  • PDH/PDL, Asia Range (H/L), EQH/EQL คือกอง SL ตามธรรมชาติ
  • พฤติกรรมซ้ำ: Sweep → BOS ก่อนเลือกทิศจริง
  • อย่าไล่ตาม sweep — รอเข้าใน value หลังการกวาด
Process (จับมือทำ)
  1. ขีด PDH/PDL + Asia Range ก่อนเสมอ
  2. หา EQH/EQL และ “เทียบฝั่ง” กับ Bias
  3. รอ sweep เก็บ stop → แล้วค่อยหาจุดเข้าใน FVG/OB
Why It Matters

ตลาดต้องใช้ สภาพคล่อง (Stop/Limit) เพื่อเคลื่อนแรง — จุดสูงต่ำที่ชัดเจนจึงเป็น “ถังเชื้อเพลิง” ก่อนวิ่งจริง

Reflection: “คุณเห็น sweep เกิดบ่อยในช่วงเปิด London/NY หรือไม่? หลัง sweep → อะไรคือหลักฐานว่าสมาร์ตมันนี่เปลี่ยนมือ?”

CH03 — Order Block & FVG Mastery

Objective

เข้าใจวิธีใช้ Order Block และ Fair Value Gap (FVG) เพื่อระบุ "โซนที่ตลาดยังมีแรงเหลือ" เป้าหมายคือระบุว่า OB ไหนยัง “สด” (valid) และ FVG ไหนเป็น value ที่สมดุลที่สุด

Process (จับมือทำ)
  1. เริ่มจากโครงสร้าง BOS ล่าสุด เพื่อหาทิศทาง (Bias)
  2. มองย้อนกลับหาช่วงแท่งก่อน BOS — แท่งสุดท้ายที่สร้างแรงคือ OB
  3. หาช่องว่างระหว่างแท่ง (FVG) หลัง BOS นั้น
  4. ใช้ 50% ของ FVG หรือ OB-body เป็น “Value Zone” สำหรับการรอย่อเข้า
  5. หาก OB ซ้อนใน FVG → โอกาสเป็นจุด Reverse สูง
What to Observe & Why
  • OB ที่แท่ง “displacement candle” มี volume สูง = OB แท้
  • FVG ที่ราคา “กลับมาแตะครึ่งหนึ่ง” แล้ว reject → แสดงว่าแรงสมดุลกลับมา
  • Zone ที่มีทั้ง OB และ FVG ทับกัน = Confluence Zone
Common Mistakes
  • เข้า OB ที่หมดแรง (ผ่าน retest ไปแล้ว)
  • ไม่ดู FVG ซ้อน OB — พลาดจุดที่ RR ดีกว่า
  • ใช้ OB/FVG โดยไม่อิง Bias (สวนเทรนด์)
Reflection: “OB ไหนที่ยังมีแรง? ให้ดูแท่งที่เกิด BOS ถัดไปตอบแทน — ถ้ายังใช้ OB เดิมอยู่ = แรงยังไม่หมด”

CH04 — Fibonacci & Value Alignment

Objective

เรียนรู้การใช้ Fibonacci Retracement อย่างถูกต้อง เพื่อหาจุดย่อ (Discount) และจุดขาย (Premium) เชื่อมโยงกับ OB/FVG เพื่อระบุโซนที่ “ตลาดต้องการกลับเข้าสมดุล”

Process (จับมือทำ)
  1. ถ้ามองขา Buy → กางจาก Swing Low → Swing High
  2. ถ้ามองขา Sell → กางจาก Swing High → Swing Low
  3. Zone สำคัญ: 50.0%, 61.8%, 78.6%
  4. หาก OB/FVG อยู่ในโซนเดียวกับระดับ Fib → คือ Confluence Value Zone
  5. ตั้ง TP1 ที่ Swing High/Low ล่าสุด, TP2 ที่ระดับ FVG ครบ, TP3 ที่ 1.272 extension
Common Mistakes
  • กาง Fib ผิดทิศ (สวนโครงสร้าง)
  • ใช้ 61.8 โดยไม่ดู OB/FVG ซ้อน — ทำให้โดน stop ก่อนวิ่ง
  • ลืมดู Liquidity ฝั่งตรงข้าม → ไม่รู้ว่าราคาอาจ sweep ก่อนเข้า zone
Reflection: “ระดับไหนคือจุด Discount จริง? — ดู OB/FVG ในบริเวณนั้นประกอบเสมอ”

CH05 — Multi-Timeframe Confirmation

Objective

เชื่อมโยง Timeframe ทั้งหมดเข้าด้วยกัน (4H → 1H → 15m → 5m → 1m) เพื่อยืนยัน Bias, Value Zone และจุดเข้า Exit อย่างมีระบบ

Process (จับมือทำ)
  1. 4H: หาทิศทางหลัก + โครงสร้างใหญ่ (BOS, CHoCH)
  2. 1H: หาโซน OB/FVG ที่ตรงกับ Bias
  3. 15m: ดู sweep และยืนยัน micro BOS
  4. 5m–1m: หาจุดเข้าแบบ “lower-timeframe trigger”
Observation
  • ยืนยันเทรนด์จาก Timeframe ใหญ่ก่อนเสมอ
  • Micro BOS คือตัวกระตุ้น entry ที่ปลอดภัยกว่า
  • ถ้าแต่ละ TF ไม่สอดคล้องกัน → ยังไม่พร้อมเข้าเทรด
Reflection: “เมื่อ TF เล็กบอกว่าเข้าได้ แต่ TF ใหญ่ยังสวนทิศ — คุณจะเลือกเชื่อใคร?”

CH06 — Session Map & Timing

Objective

เข้าใจจังหวะของตลาดในแต่ละช่วงเวลา เพื่อรู้ว่า “แรง” จะเกิดเมื่อใด ตลาดไม่เคลื่อนไหวตลอดเวลา — แต่จะระเบิดเมื่อถึง Session สำคัญ

Session Profiles
  • Asia Session: ช่วงสะสม (range) — เหมาะดู sweep และ liquidity trap
  • London Session: จุดเริ่ม “จริงจัง” ของวัน — เกิด displacement แรก
  • New York Session: ช่วง follow หรือ reversal หลัง news
Timing Framework
  1. ตีกรอบเวลา Asia (00:00–06:00 UTC)
  2. ตีกรอบ London (07:00–12:00 UTC)
  3. ตีกรอบ NY (13:00–17:00 UTC)
  4. สังเกต sweep เกิดช่วงเปิด session
  5. ใช้ OB/FVG ของ session ก่อนหน้า เป็น reference zone
Common Mistakes
  • เทรดช่วง Asia โดยไม่รู้ว่ามันคือช่วง “สะสม”
  • ไม่รู้ว่า news ของ NY สามารถล้างทั้งวันได้
  • เข้าเทรดช้าเกินหลัง session move จบไปแล้ว
Reflection: “ทุกครั้งที่คุณเปิดกราฟ ลองถามตัวเอง — ตอนนี้ตลาดอยู่ใน Session ไหน และแรงกำลังจะเริ่มหรือกำลังจะหมด?”

CH07 — Entry Confirmation & Smart Entry Models

Objective

ยืนยันจุดเข้าให้ “คมและซื่อสัตย์กับแผน” ด้วยทริกเกอร์ระดับต่ำ (1–5m) ที่สอดคล้องกับ Bias/Value/Timing จาก TF ใหญ่

Smart Entry Models (SEM)
  1. SEM-1: Sweep → micro BOS → Return-to-Value
    รอกรวาดสภาพคล่อง (sweep) บน 1–5m → เกิด micro BOS สวน → ให้เข้าที่ 50% FVG หรือ OB body
  2. SEM-2: OB-in-FVG
    เมื่อ OB ซ้อนใน FVG ที่เกิดจาก displacement → วางคำสั่ง limit ที่ 50% FVG หรือ OB open
  3. SEM-3: Session Killzone Trigger
    ระบุตำแหน่ง sweep ใน Asia → ทริกเกอร์ micro BOS ที่ London Open/NY Open
Checklist: ยืนยันก่อนกดเข้า
  • ✅ Bias จาก 4H/1H ชัดเจน
  • ✅ อยู่ใน Value Zone (Discount สำหรับ Buy / Premium สำหรับ Sell)
  • ✅ เกิด micro BOS จริง (ปิดแท่ง)
  • ✅ มี RR ≥ 1:3 (ชัดเจนถึง TP2/ตำแหน่งสระถัดไป)
  • ✅ ข่าวเสี่ยงแรง? ถ้ามี → ลดขนาดไม้/เลี่ยง
Reflection: “ถ้ายกเลิกเงื่อนไขหนึ่งข้อออก — ดีลนี้ยังคุ้มค่าหรือไม่?”

CH08 — Risk Management & Journal System

Objective

อยู่รอดก่อน ค่อยเก่ง — ใช้การจัดการความเสี่ยงแบบมีกฎชัดเจน และจดบันทึกเพื่อยกระดับการตัดสินใจ

Risk Rules
  • ความเสี่ยงต่อดีล: 0.25–0.5% (ทุนเริ่ม $15 ใช้ความเสี่ยงต่ำสุด)
  • Max Daily Loss: 1–1.5% (หยุดทันทีเมื่อแตะ)
  • SL: หลังสวิง/หลัง OB ไม่ใช้ตัวเลขตายตัว
  • TP1: 1–2R / TP2: opposing OB / TP3: liquidity pool ถัดไป
  • Compounding เฉพาะช่วงชนะต่อเนื่อง (SNF ≥ 7/10)
Journal (จับมือทำ)
  1. บันทึก: เหตุผลเข้า (Bias/Value/Timing/Confirm)
  2. แผน SL/TP และการบริหารระหว่างถือ
  3. ภาพกราฟก่อนเข้า–หลังกดปิด
  4. บทเรียน: ความคิด/อารมณ์/ความผิดพลาด
Tip: “ชนะด้วยวินัยมากกว่าความอัจฉริยะ”

CH09 — Trade Setup Master (SNF Model + Workflow)

SNF Scoring (0–10 คะแนน/หัวข้อ)
  • Bias: โครงสร้าง TF ใหญ่ยืนยัน (BOS/CHoCH)
  • Value: อยู่ใน Discount/Premium + OB/FVG ซ้อน
  • Timing: Session/Killzone เหมาะ หรือหลังข่าวสำคัญ
  • Confirmation: micro BOS / Pattern Trigger

เข้าดีลเมื่อ คะแนนรวม ≥ 28/40

Master Workflow (สเต็ปปฏิบัติจริง)
  1. 4H → 1H: กำหนด Bias + วงโซน OB/FVG หลัก
  2. 15m: มองหา inducement/sweep ในเส้นทางสู่โซน
  3. 5m → 1m: รอ micro BOS → เล็ง 50% FVG/OB
  4. คำนวณ RR/วาง SL/TP / ขนาดไม้ตามกฎความเสี่ยง
  5. เข้าและบริหาร: partial TP / move to BE เมื่อผ่านจุด
  6. บันทึกภาพ/เหตุผล/ผลลัพธ์ → สรุปบทเรียน

CH10 — Psychological Framework & Self Review

Emotional Anchors
  • FOMO → ถามว่า “BOS แล้วหรือยัง?”
  • Overconfidence → เช็ค SNF Score ใหม่
  • Anchoring → อย่าผูกใจราคาเดิม ใช้โครงสร้างนำ
Self Review (หลังเทรด)
  1. ดีลนี้ชนะเพราะโครงสร้าง หรือเพราะฟลุค?
  2. ถ้าแพ้—แพ้เพราะเงื่อนไขข้อไหนไม่ครบ?
  3. สิ่งที่จะทำต่างออกไปครั้งหน้า?

CH11 — Deep Dive: XAUUSD • EURUSD • USDJPY

XAUUSD (ทองคำ)
  • ไวต่อข่าว Fed/เงินเฟ้อ — มัก sweep PDH/PDL ก่อนเลือกทิศ
  • ชอบเคลียร์ FVG ใหญ่แล้วค่อยไปต่อ
  • Timing เด่น: London Open / NY news
EURUSD
  • โครงสร้างค่อนข้าง “สะอาด” — อ่าน BOS ได้ชัด
  • Confluence ดีเมื่อ OB/FVG อยู่ใกล้ 50–61.8%
  • London–NY overlap มักให้ move ที่มีคุณภาพ
USDJPY
  • แนวโน้มยืดได้ไกลเมื่อ BOJ/นโยบายชัดเจน
  • ระวัง spike ช่วงข่าว — ใช้ขนาดไม้เล็กลง
  • หาจังหวะต่อเทรนด์ด้วย OB-in-FVG
